myFlix

myFlix will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies!!

The table below describes each endpoint and how to format requests to the API correctly

Description URL Endpoints HTTP Method Request body data format Response body data format
Return a list of ALL movies to the user /movies GET None A JSON object holding data about all movies, containing a description, the genre, director, and image URL. Example:

{ “Title”: “Silence of the Lambs”, “Description”: “A young F.B.I. cadet must receive the help of an incarcerated and manipulative cannibal killer to help catch another serial killer, a madman who skins his victims.”, “Genre”: “Thriller”, “Director”: “Jonathan Demme”, “Image URL”: silence_of_the_lambs.png}

Return data (description, genre, director, image URL, whether it’s featured or not) about a single movie by title to the user /movies/[title] GET None A JSON object holding data about a single movie by title. Example:

See Above
Return data about a genre (description) by name/title (e.g., “Thriller”) /genres/[genre] GET None A JSON object containing a description of particular genre. Example:

{“Description”: “A thriller is a genre of literature, film, and television whose primary feature is that it induces strong feelings of excitement, anxiety, tension, suspense, fear, and other similar emotions in its readers or viewers—in other words, media that thrills the audience.”}

Return data about a director (bio, birth year, death year) by name /directors/[director] GET None A JSON object containing data about a single director by name. Example:

{“Director”: “Jonathan Demme”, “Bio”: “American director, producer, and screenwriter, etc.”, “Born”: “February 22nd, 1944”, “Died”: “April 26th, 2017”}

Allow new users to register /users POST A JSON object containing data about the new user. Example:

{"Username":"janesmith","Password":"password123","email":"jsmith@gmail.com","Date of Birth":"January 1, 1985"}

A JSON object containing data about the user that was added, including an ID. Example:

{“Id”:1, “Username”:”janesmith”, “Password”:”password123”, “email”:”jsmith@gmail.com”, “Date of Birth”:"January 1, 1985"}

Allow users to update their user info (username, password, email, date of birth) /users/[name]/[info] PUT None A text message indicating the user information was updated
Allow users to add a movie to their list of favorites /users/[name]/[favorites] POST None A text message indicating the movie title was added to the user’s favorites
Allow users to remove a movie from their list of favorites /users/[name]/[favorites] DELETE None A text message indicating the movie titles was removed from the user’s favorites
Allow existing users to deregister /users/[name] DELETE None A text message indicating the user was deregistered
